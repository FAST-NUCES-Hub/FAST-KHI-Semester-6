Lab 7 — Constraint Satisfaction Problems

Concepts (1-line each)

  • CSP = problem defined by 3 things: X (variables), D (domains), C (constraints). Goal: assign values from D to X without breaking any C.
  • Consistent (legal) assignment: doesn't violate any constraint (e.g. Ludo players each get a different colour).
  • Complete assignment: every variable has a value (e.g. fully solved Sudoku).
  • Partial assignment: only some variables assigned (e.g. half-filled Sudoku).
  • Discrete domain: infinite, multiple variables can map to same state.
  • Finite domain: one specific domain per variable, continuous states.

Constraint types — memorise the names

TypeMeaningOR-Tools call
UnaryRestricts ONE variablemodel.add(x != 3)
BinaryTwo variables relatedmodel.add(x != y)
GlobalWhole set, no pairwise patternmodel.add_all_different([x,y,z])
LinearInteger variables in linear formmodel.add(2*x + 3*y <= 10)
Non-linearNon-linear integer relationmodel.add(x*y == z)

Consistencies

  • Node consistency: every value in domain satisfies unary constraints.
  • Arc consistency: every value in domain satisfies binary constraints with a related variable.
  • Path consistency: arc consistency extended over a third variable.
  • k-consistency: stronger generalisation over k variables.
  • Constraint propagation: special inference that prunes illegal values from domains using these consistencies.

# ===== 1. VARIABLES — change names + domain =====
n = 3                                       # <-- swap domain size

Print all solutions (callback class — memorise the skeleton)

class VarArraySolutionPrinter(cp_model.CpSolverSolutionCallback):
    def __init__(self, variables):
        cp_model.CpSolverSolutionCallback.__init__(self)
        self.__variables = variables
        self.__solution_count = 0
    def on_solution_callback(self):
        self.__solution_count += 1
        for v in self.__variables:
            print(f"{v}={self.value(v)}", end=" ")
        print()
    @property
    def solution_count(self): return self.__solution_count

solver = cp_model.CpSolver()
printer = VarArraySolutionPrinter([x, y, z])
solver.parameters.enumerate_all_solutions = True
solver.solve(model, printer)
print(f"Found {printer.solution_count} solutions")

Lab 8 — Alpha-Beta Pruning

Concepts

  • Game Theory: models strategic interactions; one player wins, the other loses.
  • Adversarial Search: agents have conflicting goals (chess, tic-tac-toe).
  • Game tree elements: Player(s), Actions(s), Result(s,a), Terminal-Test(s), Utility(s,p) (= +1 win / 0 draw / −1 lose).
  • Max: tries to maximise the value. Min: tries to minimise it.
  • α (alpha): best (highest) value Max is currently guaranteed → starts at -∞. Lower bound.
  • β (beta): best (lowest) value Min is currently guaranteed → starts at +∞. Upper bound.
  • Pruning rule: if β ≤ α, stop exploring further children — they can't change the decision.
  • Alpha-Beta gives the same answer as Minimax but faster (less nodes explored).
  • It uses DFS (Depth-First Search), not BFS.

🔑 Alpha-Beta Master Template (game tree)

import math

def alpha_beta(node, depth, alpha, beta, maximizing):
    # Leaf or depth limit -> return static value
    if depth == 0 or not node.children:
        return node.value

    if maximizing:
        value = -math.inf
        for child in node.children:
            value = max(value, alpha_beta(child, depth-1, alpha, beta, False))
            alpha = max(alpha, value)
            if beta <= alpha:
                print("Pruned at:", child.value)
                break                                # β cut-off
        node.minmax_value = value
        return value
    else:
        value = math.inf
        for child in node.children:
            value = min(value, alpha_beta(child, depth-1, alpha, beta, True))
            beta = min(beta, value)
            if beta <= alpha:
                print("Pruned at:", child.value)
                break                                # α cut-off
        node.minmax_value = value
        return value


class Node:
    def __init__(self, value=None):
        self.value = value
        self.children = []
        self.minmax_value = None

# ===== Build the tree (swap shape per question) =====
root = Node('A')
b1 = Node('B'); b2 = Node('C'); root.children = [b1, b2]
d  = Node('D'); e  = Node('E'); b1.children = [d, e]
f  = Node('F'); g  = Node('G'); b2.children = [f, g]
d.children = [Node(2), Node(3)]
e.children = [Node(5), Node(9)]
f.children = [Node(0), Node(1)]
g.children = [Node(7), Node(5)]

# ===== Run from root with α=-∞, β=+∞, Max starts =====
best = alpha_beta(root, depth=3, alpha=-math.inf, beta=math.inf, maximizing=True)
print("Best value at root:", best)
How to memorise the recursion:
  1. Base case: leaf or depth=0 → return the value.
  2. If MAX: start v = -∞, loop children, v = max(v, recurse), then α = max(α, v), then check β ≤ α → break.
  3. If MIN: mirror — start v = +∞, use min, update β = min(β, v), same prune check.
The check β ≤ α is the same in both — only v, the comparator, and which bound updates change.

Chess Game with Alpha-Beta (memorize this exact pattern)

Uses the python-chess library. The trick: same α-β recursion, but moves are children and a piece-value evaluation function replaces the leaf value.

import chess
import math

# --- Piece values for the evaluation function ---
PIECE_VALUES = {
    chess.PAWN:   1,
    chess.KNIGHT: 3,
    chess.BISHOP: 3,
    chess.ROOK:   5,
    chess.QUEEN:  9,
    chess.KING:   0,
}

def evaluate(board):
    # White - Black material count
    score = 0
    for piece_type in PIECE_VALUES:
        score += len(board.pieces(piece_type, chess.WHITE)) * PIECE_VALUES[piece_type]
        score -= len(board.pieces(piece_type, chess.BLACK)) * PIECE_VALUES[piece_type]
    return score

def alpha_beta(board, depth, alpha, beta, maximizing):
    if depth == 0 or board.is_game_over():
        return evaluate(board)

    if maximizing:
        value = -math.inf
        for move in board.legal_moves:
            board.push(move)
            value = max(value, alpha_beta(board, depth-1, alpha, beta, False))
            board.pop()
            alpha = max(alpha, value)
            if beta <= alpha: break
        return value
    else:
        value = math.inf
        for move in board.legal_moves:
            board.push(move)
            value = min(value, alpha_beta(board, depth-1, alpha, beta, True))
            board.pop()
            beta = min(beta, value)
            if beta <= alpha: break
        return value

def best_move(board, depth):
    best, best_val = None, -math.inf
    for move in board.legal_moves:
        board.push(move)
        val = alpha_beta(board, depth-1, -math.inf, math.inf, False)
        board.pop()
        if val > best_val:
            best_val, best = val, move
    return best

# ===== Game loop: AI vs Human =====
board = chess.Board()
while not board.is_game_over():
    print(board)
    if board.turn == chess.WHITE:                  # AI move
        move = best_move(board, depth=3)
        print("AI plays:", move)
        board.push(move)
    else:                                          # Human move
        uci = input("Your move (e.g. e7e5): ")
        board.push_uci(uci)

print("Result:", board.result())
Why this is the only chess code you need: the same alpha_beta function works for any board game — just swap (a) the legal moves enumeration, (b) evaluate(), and (c) the make/undo move. For tic-tac-toe just replace board operations.

Coin-pick game with α-β (Task 2 of lab)

import math

def alpha_beta(coins, alpha, beta, maximizing):
    if not coins:
        return 0
    if maximizing:
        v = -math.inf
        for pick in [coins[0], coins[-1]]:
            rest = coins[1:] if pick == coins[0] else coins[:-1]
            v = max(v, pick + alpha_beta(rest, alpha, beta, False))
            alpha = max(alpha, v)
            if beta <= alpha: break
        return v
    else:
        v = math.inf
        for pick in [coins[0], coins[-1]]:
            rest = coins[1:] if pick == coins[0] else coins[:-1]
            v = min(v, alpha_beta(rest, alpha, beta, True))
            beta = min(beta, v)
            if beta <= alpha: break
        return v

Lab 9 — Bayesian Networks

Concepts

  • Bayesian Network = Directed Acyclic Graph (DAG) of variables + a Conditional Probability Table (CPT) at every node. Used for prediction, diagnostics, anomaly detection under uncertainty.
  • 2 components: (1) DAG = structure of dependencies, (2) CPTs = the actual numbers.
  • Nodes = random variables (observable like symptoms, or hidden like disease).
  • Edges A → B mean B is conditionally dependent on A.
  • Root nodes = no parents (priors).
  • Leaf nodes = no children (often the observed outputs).
  • Parent nodes = nodes pointing into the current one. CPT must list a probability for every combination of parents' values.

🔑 Bayesian Network Master Template (Burglary–Alarm flavour)

Every BN question is the same skeleton. Just swap the edges and CPDs.

from pgmpy.models import DiscreteBayesianNetwork
from pgmpy.factors.discrete import TabularCPD
from pgmpy.inference import VariableElimination

# ===== 1. STRUCTURE — list of (parent, child) edges =====
model = DiscreteBayesianNetwork([
    ('Burglary',   'Alarm'),
    ('Earthquake', 'Alarm'),
    ('Alarm',      'JohnCalls'),
    ('Alarm',      'MaryCalls'),
])

# ===== 2. CPDs — root nodes have NO evidence =====
cpd_b = TabularCPD('Burglary',   2, [[0.999], [0.001]])
cpd_e = TabularCPD('Earthquake', 2, [[0.998], [0.002]])

# Child node with 2 parents -> 2*2 = 4 columns
# Column order: (B=F,E=F), (B=F,E=T), (B=T,E=F), (B=T,E=T)
cpd_a = TabularCPD('Alarm', 2,
    values=[[0.999, 0.71, 0.06, 0.05],   # Alarm=False
            [0.001, 0.29, 0.94, 0.95]],  # Alarm=True
    evidence=['Burglary', 'Earthquake'],
    evidence_card=[2, 2])

cpd_j = TabularCPD('JohnCalls', 2,
    values=[[0.3, 0.9], [0.7, 0.1]],
    evidence=['Alarm'], evidence_card=[2])

cpd_m = TabularCPD('MaryCalls', 2,
    values=[[0.2, 0.99], [0.8, 0.01]],
    evidence=['Alarm'], evidence_card=[2])

# ===== 3. ATTACH + VERIFY =====
model.add_cpds(cpd_b, cpd_e, cpd_a, cpd_j, cpd_m)
assert model.check_model()

# ===== 4. INFERENCE =====
inference = VariableElimination(model)
result = inference.query(variables=['Burglary'],
                         evidence={'JohnCalls': 1, 'MaryCalls': 1})
print(result)

⚠️ The 3 CPD rules you MUST remember

  1. Number of rows = number of states the node has (variable_card).
  2. Number of columns = product of parents' cards (e.g. 2 binary parents → 4 columns).
  3. Each column must sum to 1 (it's a probability distribution).

Lab 9 — Markov Models

Concepts

  • Markov property = "memoryless": next state depends only on the current state, not on history.
  • Two types: Observable Markov Models (states known) and Hidden Markov Models, HMMs (states inferred from observable outputs).
  • Transition matrix = NxN matrix; row i, col j = P(next = j | current = i). Each row sums to 1.
  • The marble-bag example: 50/50 because the bag is reset every draw (with replacement).

🔑 Markov Master Template

import numpy as np

# ===== 1. STATES + TRANSITION MATRIX (rows must sum to 1) =====
states = ["Sunny", "Cloudy", "Rainy"]
P = np.array([
    [0.7, 0.2, 0.1],     # from Sunny
    [0.3, 0.4, 0.3],     # from Cloudy
    [0.2, 0.3, 0.5],     # from Rainy
])

# ===== 2. SIMULATION =====
def simulate(start, steps):
    seq = [start]
    cur = states.index(start)
    for _ in range(steps):
        cur = np.random.choice(len(states), p=P[cur])
        seq.append(states[cur])
    return seq

sequence = simulate("Sunny", 10)
print(" -> ".join(sequence))

# ===== 3. P(at least 3 rainy days over 10 days) — Monte Carlo =====
trials = 10000
hits = 0
for _ in range(trials):
    seq = simulate("Sunny", 10)
    if seq.count("Rainy") >= 3:
        hits += 1
print("P(>=3 rainy) ≈", hits / trials)

Lab 10 — Supervised Learning

Concepts (1-line)

  • Supervised learning: trained on labelled data (X, y). Goal: learn mapping X → y.
  • Two sub-types: Classification (categorical output: spam/not, yes/no) and Regression (continuous output: price, age).
  • Underfitting: model too simple; bad on both train and test.
  • Overfitting: memorises training data; great on train, bad on test. Fix: train-test split + cross-validation.
  • K-Fold CV: split into K folds; train on K-1, test on 1; repeat K times; average accuracy.
  • LOOCV: extreme case — K = N samples (one sample per fold).
  • Confusion matrix: TP / FP / TN / FN for binary classification.
  • ROC curve: TPR (y) vs FPR (x). AUC near 1 = great, near 0.5 = random.

SVM Kernels

  • Linear: K(x,x') = xᵀx' — for linearly separable data.
  • Polynomial: K(x,x') = (xᵀx' + c)d.
  • RBF (Radial Basis Function): K(x,x') = e−γ‖x−x'‖² — most common, handles non-linear.

Decision Tree terms

  • Root node: first split.
  • Splitting: dividing a node into sub-nodes.
  • Decision node: a node that splits further.
  • Leaf node: terminal — gives the prediction.
  • Branch / sub-tree: subsection of the tree.
  • Steps: pick best attribute (using Attribute Selection Measure / ASM) → split → recurse on subsets.

Lab 11 — Unsupervised Learning (K-Means)

Concepts

  • Unsupervised learning: no labels. Algorithm finds hidden structure in X alone.
  • K-Means: groups data into K clusters by minimising sum of squared distances to each cluster centre (= inertia).
  • Sensitive to feature scales → always scale with StandardScaler first.
  • Elbow Method: plot inertia vs K for K = 1..10; pick the K at the "elbow" bend (where adding more clusters stops giving big improvements).
  • No "true labels" so a confusion matrix isn't directly applicable — instead profile clusters (mean of features per cluster) and list members.
  • K-Fold CV for clustering isn't for accuracy — it checks stability (how consistent inertia is across folds).

🔑 K-Means Master Template (mirrors the Country-data notebook)

import pandas as pd, numpy as np
import matplotlib.pyplot as plt, seaborn as sns
from sklearn.preprocessing import StandardScaler
from sklearn.cluster      import KMeans
from sklearn.model_selection import KFold

# ===== 1. LOAD =====
data = pd.read_csv("Country-data.csv")

# ===== 2. EDA (the 4 must-haves) =====
print(data.head())
print(data.info())
print(data.isnull().sum())
print(data.describe())

# ===== 3. PREPROCESS — drop non-numeric, then scale =====
X = data.drop('country', axis=1)            # <-- swap 'country' for ID column
X_scaled = StandardScaler().fit_transform(X)

# ===== 4. ELBOW METHOD — find best K =====
inertia = []
k_range = range(1, 11)
for k in k_range:
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X_scaled)
    inertia.append(km.inertia_)

plt.figure(figsize=(10, 6))
plt.plot(k_range, inertia, marker='o')
plt.title("Elbow Method for Optimal K")
plt.xlabel("Number of clusters (K)")
plt.ylabel("Inertia")
plt.xticks(k_range); plt.grid(True); plt.show()

# ===== 5. FIT WITH OPTIMAL K =====
optimal_k = 3                              # <-- pick from elbow visually
kmeans = KMeans(n_clusters=optimal_k, random_state=42, n_init=10)
data['Cluster'] = kmeans.fit_predict(X_scaled)

# ===== 6. INSPECT =====
print(data.head())
print(data['Cluster'].value_counts().sort_index())  # size of each cluster

# ===== 7. PROFILE — mean of original features per cluster =====
cluster_means = data.groupby('Cluster')[X.columns].mean()
print(cluster_means)

# ===== 8. LIST MEMBERS =====
for cid in sorted(data['Cluster'].unique()):
    members = data[data['Cluster'] == cid]['country'].tolist()
    print(f"Cluster {cid} ({len(members)}): {members}")

# ===== 9. K-FOLD STABILITY CHECK =====
kf = KFold(n_splits=5, shuffle=True, random_state=42)
fold_inertias = []
for tr, te in kf.split(X_scaled):
    km = KMeans(n_clusters=optimal_k, random_state=42, n_init=10)
    km.fit(X_scaled[tr])
    fold_inertias.append(km.inertia_)
print("Avg inertia:", np.mean(fold_inertias))
print("Std inertia:", np.std(fold_inertias))

Common Mistakes & Tips

  • CPT columns wrong: column count = product of parents' cards. 2 binary parents → 4 columns. 3 binary parents → 8 columns. Each column sums to 1.
  • State indexing in pgmpy: by default 0 = "False/first", 1 = "True/second". So JohnCalls=1 means John called.
  • Order of CPT columns for evidence=['B','E']: (B=0,E=0), (B=0,E=1), (B=1,E=0), (B=1,E=1).
  • Markov rows in transition matrix must sum to 1, not columns.
  • Always set random_state=42 in train_test_split / KFold / KMeans for reproducible answers.
  • SVM / Logistic Regression need scaling first; tree-based models don't.
  • K-Means: n_init=10 avoids the future-warning.
  • OR-Tools constraint forgotten: a CSP question is wrong if you forgot the global constraint (e.g. add_all_different in N-Queens / Sudoku).
  • α-β prune check = if beta <= alpha: break. Same in both Max and Min branches.
  • Don't forget plt.show() after plots.
  • X.iloc vs X[]: in K-Fold, if X is a NumPy array use X[idx]; if it's a DataFrame use X.iloc[idx].
